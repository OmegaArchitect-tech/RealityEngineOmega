Fantasy Analyzer & Anomaly Injector Toolkit
Bridging Conceptual Fiction and Mathematical Physics
================================================================================

OVERVIEW
This toolkit bridges the gap between conceptual fiction and mathematically rigorous game engine physics. It acts as the pipeline between Large Language Model (LLM) reasoning and Unreal Engine 5 C++ execution. It consists of two primary components: The Fantasy Analyzer (an LLM System Prompt) and the Anomaly Injector (a JSON-to-C++ pipeline).

COMPONENT 1: THE FANTASY ANALYZER (LLM PROMPT)
The Analyzer is a master system prompt designed to be deployed in top-tier LLMs. It acts as an absolute, mathematically rigid physics adjudicator. Instead of evaluating fictional lore, it calculates the joule energy, waste heat, and spacetime impact of any spell, item, or creature.

Features:
* Ruthless thermodynamic auditing of fictional concepts.
* Calculates the physical cost of anomalies (e.g., freezing the caster to absolute zero to satisfy the First Law).
* Generates a clinical "Crash Report" and assigns a Reality Grade (S-Tier to F-Tier).
* Offers a "Calibration Matrix" to physically "nerf" impossible magic into engine-safe mechanics.

How to Use:
1. Copy the `Reality_Engine_Analyzer_Master_Prompt.txt` text and paste it as the System Instructions in your chosen LLM.
2. The Analyzer will initialize and ask for three strict parameters:
   - Fantasy: The source of the fiction (e.g., The Witcher, Star Wars).
   - Format: The medium of the story (e.g., Video Game, Book).
   - Target: The specific anomaly being tested (e.g., "Spell: Fireball").
3. Provide the parameters. The LLM will generate the Crash Report and output a highly structured JSON file containing the anomaly's exact physical parameters.

COMPONENT 2: THE ANOMALY INJECTOR (JSON API)
The Injector pipeline bridges the text-based AI analysis and the compiled Unreal Engine environment. 

Features:
* Parses AI-generated JSON payloads directly into the `RealityEngineOmega` UE5 subsystem.
* Maps text outputs to hard C++ variables: Temperature_K, InternalEnergy_J, Complexity, Efficiency, and 3D Location coordinates.

How to Use:
1. Retrieve the JSON file generated by the Fantasy Analyzer (e.g., `Anomaly_Injector_Fireball.json`).
2. Place the JSON file into your designated engine data directory (e.g., `Example_Payloads_JSON`).
3. Compile and run your UE5 project with `RealityEngineOmega` active.
4. The `Anomaly_API.cpp` (when triggered in your game logic) will parse this payload, mapping the values directly to an `FThermodynamicActor` struct.
5. The engine will read the injected parameters and ruthlessly enforce thermodynamics and spacetime curvature upon the newly spawned anomaly in real-time.
