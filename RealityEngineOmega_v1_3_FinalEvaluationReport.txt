FINDING V1-2-01 | CLOSED IN V1.3 | WAS: Physics bug (TDSE kinetic operator)
--------------------------------------------------------------------------------
TITLE: TDSE Euler propagator verified correct — V1-2-01 closed

VERIFICATION_METHOD:
  Derived the correct TDSE Euler update from first principles and compared
  against the v1.3 implementation line-by-line.

DERIVATION:
  TDSE: iħ ∂Ψ/∂t = ĤΨ = [-ħ²/2m ∇² + V]Ψ
  Writing Ψ = R + iI and separating real/imaginary parts:

    ∂R/∂t = +(1/ħ)(KI + V·I)
    ∂I/∂t = -(1/ħ)(KR + V·R)

  where KR = -(ħ²/2m)∇²R  and  KI = -(ħ²/2m)∇²I

  Euler update:
    NextR[i] = R[i] + (dt/ħ)(KI + V·PsiImag[i])
    NextI[i] = I[i] - (dt/ħ)(KR + V·PsiReal[i])

V1.3_CODE_MATCHES_DERIVATION: YES (line-by-line match confirmed)

ADDITIONAL_CHECKS_PASSED:
  - d2R and d2I both computed with GridSpacing² (dx²) normalisation ✓
  - Double-buffer (NextPsiReal/NextPsiImag copied before loop) prevents
    in-place aliasing where updated i-1 values contaminate computation at i ✓
  - Dirichlet boundary conditions at endpoints (d=0 at i=0, i=N-1) — valid
    choice for a bounded 1D quantum system ✓
  - PotentialEnergyV graceful fallback to V=0 when array shorter than PsiReal ✓

SEMANTIC_NOTE:
  Changelog states "Unitarity preserved." More precisely: the Euler method is
  correct to first order and approximately unitary. True exact unitarity requires
  Crank-Nicolson or split-operator methods. For a game engine with small DeltaTime
  (dt ≈ 0.016s at 60fps), first-order Euler is appropriate. This is a naming
  precision issue in the changelog, not an error in the code.

PREVIOUS_ERRORS_FROM_V1.2_ELIMINATED:

  ✓ Imaginary Laplacian (d2I) now computed and applied

  ✓ Phase rotation approach replaced with correct Euler integration
