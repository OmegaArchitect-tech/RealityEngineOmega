================================================================================

         FTickableGameObject as a second base class.

ROOT_CAUSE:
  class MYPROJECT2_API URealityEngineOmega : public UWorldSubsystem, public FTickableGameObject

  UWorldSubsystem is a UObject with its own tick infrastructure.

  Compiler sees nested class with same name as enclosing class → C3769 fatal.
  Every subsequent line produces cascade errors (C2143, C2504, C2440, etc.).

COMPILER_ERROR_PRIMARY:
  error C3769: 'URealityEngineOmega': a nested class cannot have the same
  name as the immediately enclosing class

DOWNSTREAM_CASCADE_COUNT: 25+ errors, all from single root cause.

ALSO_MISSING: U prefix on class name — original declared RealityEngineOmega
              but .cpp used URealityEngineOmega (separate issue compounding the failure).

CORRECT_FIX:
  Remove FTickableGameObject entirely.
  Derive ONLY from UWorldSubsystem.
  Add bCanEverTick = true in Initialize().
